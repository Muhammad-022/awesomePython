# Day 67: Expense Tracker Project

## Concept
An Expense Tracker helps manage daily spending.

This project uses:
- File/Database to store expenses
- OOP (Object-Oriented Programming) for structure

## Features
- Add expense
- View all expenses
- View total summary

## Data Handling
- Expenses stored in a file (JSON)
- Each expense includes:
  - Title
  - Amount

## OOP Structure

### Class: ExpenseTracker
Handles all operations:
- Add expense
- Save/load data
- Calculate total

## How It Works

1. Add Expense  
   - Input title and amount  
   - Save to file  

2. View Expenses  
   - Display all stored expenses  

3. View Summary  
   - Calculate total spending  

## Why This Project?
- Real-world use case
- Combines file handling + OOP
- Builds logic and structure

## Future Improvements
- Add categories (food, travel, etc.)
- Monthly reports
- Database integration (SQLite)
