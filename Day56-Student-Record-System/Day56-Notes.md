# Day 56: OOP Project – Student Record System

## Concept
This project combines multiple OOP concepts with file handling to create a simple **Student Record System**.

Features include:
- Adding new student records
- Searching student records by ID
- Updating student information
- Deleting student records
- Storing all records in a file for persistence

The project uses:
- **Classes & Objects** – `Student` and `StudentManager`
- **File Handling** – saving and reading student data
- **OOP Principles** – Encapsulation, Methods, and Class Organization

## Why Use This Project?
- Practice combining **multiple OOP concepts**
- Learn to **manage data in files**
- Understand how real-world systems are structured
- Improve problem-solving skills in Python

## Key Points
- Student data is stored in a text file (`students.txt`)
- Each student has attributes: ID, Name, Age, Grade
- Manager class handles all operations: add, search, update, delete
- Modular design: easy to expand for more features
